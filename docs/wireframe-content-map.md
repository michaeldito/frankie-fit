# Frankie Fit Wireframe Content Map

## Purpose

This document translates the current product strategy into a wireframe-style content map. It does not define final visual design, but it does define:

- which major surfaces exist
- what blocks appear on each surface
- the recommended order of those blocks
- what each block is trying to communicate or drive the user to do

This document should help bridge the gap between product strategy, copywriting, UI design, and engineering.

## How To Read This

Each screen is described in a top-to-bottom layout order. For each block, this doc calls out:

- what the block is
- what it should contain
- why it exists
- what action it should encourage

## Global UX Rules

- Chat is the primary home of the product.
- Dashboard content should support Frankie, not compete with Frankie.
- Each major screen should have one clearest next action.
- The app should feel calm and focused rather than crowded.
- Frankie should be visible across the product, even outside the main chat screen.

## Public Marketing Site

## Landing Page

### Goal

Help a new visitor understand the product quickly, trust it, and take the next step.

### Primary CTA

Get early access

### Block Order

#### 1. Header

Content:

- Frankie Fit wordmark
- nav links:
  - Product
  - How It Works
  - Frankie
  - Privacy
  - Early Access
- primary CTA button

Purpose:

- establish brand
- give a simple navigation structure
- keep the signup action visible

#### 2. Hero

Content:

- headline:
  - `Log less. Understand more. Stay consistent.`
- subheadline
- primary CTA
- secondary CTA
- product mockup or illustrative product frame

Purpose:

- explain the product in one quick glance
- position Frankie Fit as more than a tracker
- move the user toward early access or learning more

#### 3. Problem Section

Content:

- headline:
  - `Most health apps ask too much and help too little.`
- short body copy
- optional visual contrast:
  - traditional tracking pain
  - Frankie Fit low-friction model

Purpose:

- make the user's frustration feel seen
- set up why this product exists

#### 4. How It Works

Content:

- section intro
- three-step explanation:
  - tell Frankie what is going on
  - Frankie organizes the signal
  - get your next best step

Purpose:

- demystify the product
- show that the experience is simple even if the system is sophisticated

#### 5. Three Pillars

Content:

- Exercise card
- Diet card
- Wellness card

Each card should include:

- short headline
- one-paragraph description
- a sentence about what the user gets from that pillar

Purpose:

- show breadth without overwhelming the visitor
- reinforce the holistic product model

#### 6. Why It Feels Different

Content:

- headline:
  - `Simple on the surface. Smarter underneath.`
- short body copy
- 4 to 5 supporting bullets

Purpose:

- explain the product magic
- position the app as thoughtfully intelligent rather than gimmicky

#### 7. Meet Frankie

Content:

- short intro to Frankie
- tone/personality description
- optional founder note about the name

Purpose:

- make the product emotionally memorable
- humanize the AI coach without making it feel cheesy

#### 8. Privacy and Trust

Content:

- privacy headline
- short trust copy
- short safety/boundary copy

Purpose:

- address the natural trust question in a wellness product
- reinforce that the product respects user data and stays in non-clinical territory

#### 9. Social Proof Placeholder

Content:

- placeholder headline
- future area for:
  - testimonials
  - founder note
  - early user feedback

Purpose:

- reserve space for future trust-building material

#### 10. Final CTA

Content:

- closing headline
- short body copy
- primary CTA
- secondary CTA

Purpose:

- convert interested visitors
- end on a simple, confident product promise

## Authenticated User App

## Global App Shell

### Purpose

Provide a simple, stable frame for the user experience.

### Layout Structure

- left navigation rail or top navigation
- primary content area
- optional compact Frankie action panel

### Main Navigation

- Chat
- Dashboard
- Profile

### Persistent Elements

- user avatar or profile access
- current goal snapshot
- compact recommended next action

### UX Goal

The user should always know:

- where they are
- what Frankie recommends next
- how to log something quickly

### Goal

Make this the fastest, most natural place to use the product.

### Block Order

#### 1. Page Header

Content:

- `Chat with Frankie`
- short helper line:
  - `Log something, ask a question, or get a plan.`

Purpose:

- frame the screen clearly

Content:

- keep Frankie as the center of the experience

#### 4. Quick Action Row

Content:

- log workout
- log meal

Purpose:

- lower the effort needed to start an interaction

#### 5. Message Composer

Content:

- free-form input
- send button
- optional placeholder text:

Purpose:

- make natural language the default mode

Send a message or log something.

## State Guidance

## Empty States

- Every empty state should reduce pressure.
- Frankie should always offer one simple next move.
- Empty states should never feel like failure.

## Low-Data States

- If a user has only a little data, show simple summaries rather than blank charts.
- Frankie should explain what more logging would unlock without sounding demanding.

## High-Activity States

- If a user is active, surface stronger insights and trend commentary.
- Keep the tone steady, not overly celebratory or intense.

## Build Notes

- Start with content hierarchy before visual styling.
- Design the shared dashboard shell once and let the three tabs inherit a common layout.
- Prioritize clarity of modules over design novelty in the earliest pass.
- Keep enough whitespace and separation so the product feels calm.

## Open Design Questions

- Should the left nav include Frankie's face or avatar persistently, or should that stay more subtle?
- Should the dashboard tabs live near the top of the page or beneath the summary block?
- How much of the landing page should feel playful versus calm and premium?
